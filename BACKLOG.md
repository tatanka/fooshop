**Ultimo aggiornamento:** 2026-03-07 (inizializzazione backlog)



## Convenzioni

- **Priorità:** P0 (critica), P1 (alta), P2 (media), P3 (bassa)
- **Complessità:** S (small), M (medium), L (large), XL (extra large)
- **Categoria:** STORE (storefront), PROD (prodotti), PAY (pagamenti), AI (intelligenza artificiale), MCP (mcp server), AUTH (autenticazione), DASH (dashboard), UI (interfaccia), BUG (bug fix), INFRA (infrastruttura), SEO (seo/marketing), API (api pubblica)
- **Date:** formato YYYY-MM-DD

---

## Come Aggiornare il Backlog

1. **Nuovo task**: Aggiungi riga nella sezione priorità appropriata con ID `CAT-XX` e data inserimento
2. **Specifica**: Se necessario, crea file `docs/specs/ID.md` con dettagli
3. **Inizio lavoro**: Sposta in sezione "In Corso" con data inizio
4. **Completamento**: Sposta in sezione "Completato" con data, **elimina file spec**
5. **Priorità**: Rivedi settimanalmente e riordina se necessario
